# Study Rules

- 이슈에 계획 작성 -> 챕터 단위로 작성
- 브랜치명 -> Chapter{ 챕터 번호 } - { 닉네임 } 
- 커밋 
  - 첫 커밋 feat - { 내용 } #{ 이슈번호 } 
  - 이후 커밋 refactor - { 내용 } #{ 이슈번호 }
- pr -> ex) Chapter1-cvMax #3
  -  내용은 공백
- 추후 회의를 통한 핃백과 Merge
- Merge 시 브랜치 리베이스
- 추가 예정

# code Rules
코드 작성시 그 과정에서 이해한 내용을 주석으로 달기.

코드 작성시 예제를 활용해서 코드를 변경한다면 그 부분도 주석으로 남겨놓고 설명.

`ch{}.{ 브랜치 이름 }.{ 실제 코드 작성 패키지 } 
ex ) ch1.cvMax.object `
